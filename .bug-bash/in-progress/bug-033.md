## Description
Add a `t` hotkey in the Command Center plugin that opens a text input for quickly adding todos. The user types one todo per line and all are created when they submit. This bypasses the existing command input which requires typing "new todo: Foo" — the `t` key should go straight to todo entry mode.

## Expected Behavior
- Press `t` in Command Center to enter todo entry mode
- Show a multi-line text input (or single-line that accepts multiple submissions)
- Each line becomes a new todo item
- Submit creates all todos at once
- Esc cancels without creating anything
- No need to type "new todo:" prefix — just the todo text

## Files Likely Involved
Unknown — agent should explore the Command Center plugin in `internal/builtin/` for how key handlers and the existing command input work, and `internal/db/` for how todos are created/stored.

## Resolution
Added `t` hotkey in the Command Center plugin that opens a dedicated textarea for quick todo entry. Each non-empty line typed becomes a new todo when the user presses ctrl+d to submit. Esc cancels without creating anything. After submission, a flash message shows the count of todos added, and the focus suggestion is refreshed.

## Files Changed
- `internal/builtin/commandcenter/commandcenter.go` — Added `addingTodoQuick` bool and `quickTodoTextArea` textarea.Model fields to Plugin struct; initialized the quick textarea in Init(); added rendering of the quick todo textarea in viewCommandTab(); added `t` to key bindings list.
- `internal/builtin/commandcenter/cc_keys.go` — Added `handleAddingTodoQuick` method (ctrl+d submits, esc cancels, each line becomes a todo); added `t` case in handleCommandTab to enter quick todo mode; added quick todo mode check in HandleKey dispatch.
- `internal/builtin/commandcenter/cc_messages.go` — Added passthrough of non-key messages to quickTodoTextArea when in quick todo mode.
- `internal/builtin/commandcenter/cc_view.go` — Added `t` shortcut to help overlay and footer hint bar.

## Choices Made
- Used a separate `quickTodoTextArea` rather than reusing the existing `todoTextArea` to avoid state conflicts with the "c" command's conversation flow.
- Used ctrl+d for submit (consistent with the existing "c" command textarea), not plain Enter, since Enter is needed for multi-line input.
- Each non-empty line becomes a separate todo via `cc.AddTodo()` + `db.DBInsertTodo()`, matching how the "c" command creates todos.
- Source is set to "manual" (the default from AddTodo) and event source is "quick" to distinguish from command-created todos.

## Uncertainties
- None significant. The implementation follows established patterns from the existing "c" command flow.
