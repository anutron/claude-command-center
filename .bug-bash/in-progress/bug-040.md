## Description
The banner top padding default is 10 (too much space). Change the default to 2.

## Expected Behavior
New installs or configs without `banner_top_padding` set should default to 2 lines of padding above the banner.

## Files Likely Involved
Unknown — agent should find where the default value is defined in `internal/config/config.go` (likely in `GetBannerTopPadding()`) and change it from 10 to 2.

## Resolution
Changed the default return value in `GetBannerTopPadding()` from 10 to 2, and updated the corresponding comment.

## Files Changed
- `internal/config/config.go` — Changed default in `GetBannerTopPadding()` from 10 to 2

## Choices Made
- Only changed the default value and comment; no other references to the old default of 10 existed in tests or elsewhere.

## Uncertainties
- None. This was a straightforward default value change with no test dependencies.
