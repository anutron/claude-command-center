# QA Checklist — Mar 14 Features

## Todo Triage Workflow

- [ ] Launch TUI, press `space` to expand todos
- [ ] Tab bar visible at top with filter categories and counts
- [ ] `tab` cycles forward through filters: Accepted > New > Review > Blocked > Active > All
- [ ] `shift+tab` cycles backward
- [ ] Switching tabs resets cursor to top
- [ ] Normal (collapsed) view shows only accepted todos with no agent session
- [ ] Status bar at bottom of todo panel shows `New(N) · Review(N) · Blocked(N) · Active(N)`
- [ ] Status bar hidden when all counts are zero
- [ ] Run `ai-cron` — external todos appear in New tab
- [ ] `y` on a New todo moves it to Accepted
- [ ] `Y` on a New todo accepts AND opens task runner
- [ ] Create todo via `c` — appears in Accepted (not New)
- [ ] Create todo via `t` (quick add) — appears in Accepted
- [ ] Launching an agent on a "new" todo auto-accepts it

## Search / Filter

- [ ] `/` activates search input
- [ ] Typing filters todos by title (case insensitive)
- [ ] Search works in both normal and expanded view
- [ ] Search applies on top of triage filter (e.g., search within "New" tab)
- [ ] `esc` clears search filter
- [ ] Filter hint shown when search is active but input dismissed

## Todo Detail View

- [ ] `enter` on a todo opens detail view
- [ ] Title bar shows "TODO #N" with display_id
- [ ] Two-column layout renders correctly
- [ ] `tab`/`shift+tab` cycles editable fields: Status, Due, ProjectDir, Prompt
- [ ] `enter` on Status cycles value (active > waiting > completed)
- [ ] `enter` on Due opens text input (YYYY-MM-DD)
- [ ] `enter` on ProjectDir cycles through known paths (or text input if none)
- [ ] `enter` on Prompt opens text input
- [ ] `c` opens command input for Claude-powered editing
- [ ] `j`/`k` navigates between todos without leaving detail view
- [ ] `x` completes todo — shows notice banner
- [ ] `X` dismisses todo — shows notice banner
- [ ] Notice banner blocks keys (except esc) for ~1s
- [ ] `esc` returns to list

## Task Runner

- [ ] `o` in detail view opens task runner
- [ ] Mode cycles: normal > worktree > sandbox (left/right arrows)
- [ ] Permission cycles: default > plan > auto
- [ ] Budget adjusts with left/right arrows ($0.50 minimum)
- [ ] Queue toggle works (auto-start on/off)
- [ ] `tab`/`shift+tab` cycles between rows
- [ ] Prompt viewport scrollable with `j`/`k`
- [ ] `p` opens Plannotator for prompt editing
- [ ] `enter` launches agent session
- [ ] Flash message confirms launch or queue

## Agent Sessions

- [ ] Launched agent shows status indicator in todo list
- [ ] Session status updates (active > review/blocked) reflected in triage tabs
- [ ] Agent completing moves todo to Review tab

## Expanded View Pagination

- [ ] `space` once: 2-column expanded view
- [ ] `space` twice: 1-column expanded view
- [ ] `space` three times: collapse back to normal
- [ ] `left`/`right` moves between columns
- [ ] At left edge of first page: no wrap
- [ ] At right edge: paginates to next page
- [ ] `left` at second page returns to first page

## Path Metadata

- [ ] `ccc paths --json` outputs learned paths with descriptions and skills
- [ ] `ccc paths --refresh-skills` refreshes skill cache
- [ ] `ccc paths --auto-describe` generates LLM descriptions for paths without one
- [ ] `ccc paths --add-rule` adds a routing rule
- [ ] `ccc todo --get <id>` outputs a single todo's details

## Incremental Sync

- [ ] First refresh processes all Granola meetings and Slack messages
- [ ] Second refresh skips already-processed items (check logs or timing)
- [ ] New meetings/messages since last sync are still picked up
